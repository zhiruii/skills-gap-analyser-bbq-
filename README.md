# Career Gap Analyser

> Analyse 10 live job postings to tell you exactly what the market wants, where your resume stands, and what to build this weekend to close the gap.

Built at 'Sup Build2026 — 12-hour hackathon, Future of Work track.

**Live demo:** [career-gap-analyser.vercel.app](https://career-gap-analyser.vercel.app)

---

## The Problem

You don't know what you lack for your target role, and the advice you get is too vague to act on. Most tools scan one job posting for keyword matches. We analyse ten at once, and we go deeper than keywords.

The output isn't "learn SQL." It's: _"Extend your X project by implementing a neural network model using TensorFlow to improve the F1 score beyond 79%"_

---

## What Makes This Different

**Live data, not model memory.**
The job market is volatile. We fetch 10 real, current job postings at query time via [Exa](https://exa.ai) — every run. The signal is current because the data is current, not because a model thinks it is.

**Verifiable receipts.**
Every claim in our output comes with a real company name, a real verbatim quote, and a clickable link to the actual job posting. That is the difference between advice you have to trust and advice you can verify.

**Depth over keywords.**
We extract not just what skills are required, but what each role expects you to *do* with them. Then we judge your resume not on keyword presence, but on whether you've demonstrated the depth the market actually expects.

**Personalisation**
Your gap analysis references your actual projects by name . Your roadmap is built on what you already have.

Recommendations are grounded in a curated dataset of 400+ real project tutorials scraped from a public GitHub repository (https://github.com/practical-tutorials/project-based-learning), filtered to your specific gaps. So instead of "learn Python," you get a specific project to build, aspecific a specific free resource, and a resume bullet you can copy the moment you're done.

## How It Works

```
User uploads resume (PDF) + enters target role
        ↓
Exa fetches 10 live job postings (60-day window, job board whitelist)
        ↓
10 parallel GPT-4o-mini calls extract { skill, context } from each posting
        ↓
Aggregation: frequency count, synonym normalisation, noise filter
        ↓
GPT-4o reads full resume PDF + aggregated market signal → gap table
        ↓
GPT-4o generates personalised roadmap grounded in existing projects
        ↓
UI renders: skills match table + expandable receipts + roadmap cards
```

Total LLM calls per request: 12 (10 extraction + 1 gap analysis + 1 roadmap)

## Key Design Decisions

### Receipts to prevent hallucination
Every skill in the output is backed by a verbatim quote from the actual Exa result, with the company name and a direct link. These receipts are extracted from the raw Exa response — not generated by any LLM. This is the core trust mechanism: a judge or user can click any receipt and verify the claim themselves.

### Personalised evidence, not generic advice
GPT-4o is explicitly prompted to cite exact metrics, technique names, and project names from the resume — not paraphrase generically. "Fine-tuned MobileNetV2 via transfer learning achieving 87.5% validation accuracy" is the kind of evidence we surface, not "candidate has machine learning experience." The roadmap is then grounded in what the specific person already has, not a generic CS student.

### Frequency filter: skills must appear in ≥3 of 10 postings
A skill appearing in 1 or 2 postings is noise — it could be one company's quirky requirement, or a low-quality posting. We only surface skills that appear in at least 3 of the 10 postings fetched. This filters noise and ensures every skill in the output represents a genuine market expectation, not a one-off.

### Job board domain whitelist on Exa
Without a whitelist, Exa returns personal portfolios, LinkedIn celebration posts, and resumes alongside actual job postings. We restrict to a curated list of job board domains (internsg.com, mycareersfuture.gov.sg, jobstreet.com, glassdoor.com, jobs.lever.co, etc.) so every result is a real job description.

### 60-day Exa date filter
30 days was too thin for niche roles — not enough postings returned to build a reliable signal. 90 days risks including stale requirements. 60 days balances recency with volume, ensuring the market signal reflects what employers are actively hiring for right now.

### Targeted project injection for the roadmap
The roadmap prompt references a pre-scraped dataset of 443 project tutorials from [practical-tutorials/project-based-learning](https://github.com/practical-tutorials/project-based-learning). However, passing all 443 entries (56,000 characters) into every prompt wastes context and drowns out relevant suggestions with noise — the LLM receives Python, JavaScript, and Go project ideas even when the gap is SQL or Cloud Platforms.

We pre-filter the dataset to only sections relevant to the detected gaps before injecting it into Prompt 3. A skill-to-section map (e.g. "machine learning" → Python section, "react" → JavaScript section) reduces the injected context to only what's applicable. For gaps the repo doesn't cover well (SQL, Tableau, CI/CD), the LLM falls back to its own knowledge — which is the correct behaviour. For gaps it does cover (Python ML projects, JavaScript web apps), the LLM now reliably cites specific tutorials with real links rather than defaulting to generic Coursera suggestions.

### GPT-4o for gap analysis and roadmap, GPT-4o-mini for extraction
Extraction (Prompt 1) is a simple structured task run 10× in parallel — GPT-4o-mini handles it reliably and cost matters at scale. Gap analysis (Prompt 2) and roadmap generation (Prompt 3) require complex cross-referencing of resume evidence, market contexts, and project ideas — GPT-4o-mini was inconsistent at this level of instruction-following. We use the stronger model where it matters.

### Per-posting deduplication in aggregation
If a single job posting mentions "Python" three times (e.g. in requirements, nice-to-haves, and tech stack), it should still count as 1/10 — not 3/10. We deduplicate skills within each posting before counting frequency, so the frequency reflects how many *distinct postings* require a skill, not how many times it appears in aggregate.

## Tech Stack

| Layer | Technology |
|---|---|
| Framework | Next.js 15 (App Router, TypeScript) |
| Frontend | React, Tailwind CSS |

---

## Security

- API keys live only in `.env.local` and Vercel environment variables — never exposed to the browser
- Rate limiting: 3 requests per minute per IP to prevent API cost abuse
- All user input is length-capped server-side (role: 200 chars, resume text: 5000 chars)
- Every LLM call has a `max_tokens` cap
- Hard spending caps set in OpenAI and Exa billing dashboards as a backstop
